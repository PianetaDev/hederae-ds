03 — Pivot tracker

Decisioni strategiche

Quando cambia una direzione fondamentale, la documentiamo qui. Serve per allineare il team e non perdere il perché delle scelte.

Pivot #1 — 8 Marzo 2026
Da "tier cliente" a "profili funzionali"

Il modello precedente classificava i progetti per dimensione/complessità del cliente (Tier 1 AGESCI = grande, Tier 4 Rara = piccolo). Questo creava confusione: un cliente piccolo può avere bisogno di un hub editoriale, uno grande solo di una vetrina. Il DS dev'essere organizzato per feature set, non per dimensione cliente.

Prima
Tier 1–4 basati su dimensione cliente
AGESCI = obiettivo finale del DS
Complessità = valore percepito
Roadmap bloccata su features lontane
Adesso
Profili A/B/C/D per feature set
Profilo A già quasi consegnabile
Profilo C è il focus (Susdef, EPOS)
Ogni profilo ha deliverable chiari

06 — Istruzioni team

Come lavorare su Hederae

Regole essenziali per contribuire in modo coerente. Per approfondimenti vedi docs/GUIDELINES.md e docs/AI-WORKFLOW.md.

Visualizzare il Design System

  1. 1Vai su pianetadev.github.io/hederae-ds
  2. 2Usa il menu a sinistra per navigare tra Token, Atoms, Molecules, pagine brand
  3. 3In alto a destra: selettore brand (Susdef / AGESCI / Rara / Corner Table) per vedere gli stessi componenti con colori/font diversi
  4. 4Usa il viewport selector per vedere mobile/tablet/desktop
Il Storybook si aggiorna automaticamente ad ogni commit su main. Il deploy impiega ~30 secondi.

Aggiungere una story Storybook

  1. 1Copia stories/_template.js nella cartella giusta (atoms / molecules / organismi / profilo)
  2. 2Imposta title con la categoria, es. '⬡ Organisms/Header'
  3. 3Ogni export è una story: la funzione render() ritorna una stringa HTML
  4. 4Usa var(--color-*), var(--font-*) per i token — mai colori hardcoded
Le stories usano @storybook/html-vite: non è Vue, è HTML puro in template literals. Tutto ciò che funziona nel browser funziona qui.

Aggiungere un nuovo brand

  1. 1Crea tokens/brand/[nome]/color.json e typography.json con i valori brand
  2. 2Crea css/themes/[nome].css con le CSS custom properties su [data-brand="nome"]
  3. 3Importa il CSS in .storybook/preview.js e aggiungi la voce in globalTypes.brand.toolbar.items
  4. 4Aggiungi Google Fonts in .storybook/preview-head.html se servono font nuovi
Il data-brand attribute viene impostato automaticamente dal decorator — non serve nient'altro.

Documentare un pivot

  1. 1Apri docs/HEDERAE-STATUS.html
  2. 2Copia il blocco .pivot-box esistente e aggiungilo sopra nella sezione #pivot
  3. 3Compila: data, titolo, descrizione del perché, before/after della direzione
  4. 4Aggiungi una voce nel log con type-pivot
Un pivot è qualsiasi decisione che cambia la direzione del DS in modo che un membro del team non si aspetterebbe. Meglio documentarne troppi che troppo pochi.
